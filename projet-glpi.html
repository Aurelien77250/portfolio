Mise en place d'un serveur GLPI sur Debian 11

Type Administration Système & ITSM
Rôle Admin Système / Support
Stack Technique Debian 11, GLPI 10, MariaDB, Apache2
Contexte et Enjeux :
Dans un environnement informatique où le nombre de postes et de licences croît, la gestion manuelle (Excel) devient source d'erreurs. Ce projet est né du besoin de centraliser l'information. L'absence d'outil dédié entraînait une méconnaissance du parc (obsolescence), une perte de temps dans le traitement des incidents et un manque de visibilité budgétaire.

2. Objectifs du Projet

L'objectif principal était de déployer GLPI (Gestionnaire Libre de Parc Informatique) pour :

  • Inventaire Automatisé : Remonter automatiquement le matériel et les logiciels.
  • Service Desk (Helpdesk) : Gérer les incidents et demandes (Conformité ITIL).
  • Gestion Financière : Suivre les contrats et cycles de vie.
  • Knowledge Base : Créer une FAQ utilisateurs et une documentation technique.

3. Architecture Technique

Déploiement sur serveur dédié pour garantir stabilité et sécurité.

  • OS : Debian (Stable) pour sa robustesse serveur.
  • Web : Apache2 + PHP (Requis pour GLPI).
  • Data : MariaDB pour le stockage.
  • Agent : GLPI Agent déployé sur les postes clients.

4. Étapes de Réalisation

Phase 1
Installation Serveur (LAMP)

Mise en place de la pile Linux, Apache, MariaDB, PHP sur Debian.

  • Sécurisation SSH et Pare-feu (UFW).
  • Création de la base de données dédiée.
  • Installation des extensions PHP requises.
Phase 2
Déploiement GLPI

Téléchargement de la version stable 10.x.

  • Configuration des droits (chown -R www-data).
  • Configuration du VHost Apache.
  • Finalisation via l'assistant web.
Phase 3
Inventaire Automatisé

Configuration de la remontée d'informations.

  • Installation du plugin GLPI Inventory.
  • Déploiement de l'agent sur un parc test (Windows/Linux).
  • Validation de l'import des données.
Phase 4
Helpdesk & Profils

Mise en production du ticketing.

  • Création profils (Tech, User, Admin).
  • Configuration des notifications SMTP (Email).
  • Catégorisation des tickets.

5. Bilan et Compétences

Ce projet a structuré le service informatique. Le passage à GLPI a réduit le temps de résolution des incidents grâce à une connaissance immédiate du matériel impacté.

Compétences validées :

  • Administration système Linux (CLI, paquets, droits).
  • Gestion de base de données (SQL, Sauvegardes).
  • Configuration Web (Apache, VHosts).
  • Processus ITIL (Incident, Demande).
